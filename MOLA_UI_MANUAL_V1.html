Mola UI

Reference Manual v0.5.43-alpha
Author: Hanan Bensho | Parametrica Labs

Design Philosophy: Seamless Integration

Mola UI is engineered to sit effortlessly on top of your existing workflows. It requires no modification to your algorithm logic and utilizes standard Grasshopper components.

The system interprets the visual organization of your canvas. If your inputs are already grouped logically, transforming them into a professional web interface requires nothing more than renaming the group. It is a "What You See Is What You Get" translation engine—preserving your parameters while elevating the user experience.

1 Getting Started

Component Location

Find all Mola UI components in Grasshopper under:

Params Tab > Mola UI Panel
Mola UI Builder Scans your canvas and generates the web interface. The core engine of the system.
Mola Styler Customizes visual theme and appearance. Connect to Builder for custom styling.
Mola Runner Launches and manages the web application independently from the Builder.

File Structure

When you run the plugin, Mola creates a folder structure next to your Grasshopper file:

  • Mola_App/ - Contains generated HTML/CSS files
    • Windows: index.html (single file with hash routing)
    • Mac: index_main.html + index_info.html (separate files)
  • Mola_Assets/ - Drop your images and icons here (.png, .svg)

Quick Start Workflow

  1. Group your Grasshopper components (Ctrl+G)
  2. Rename the group using Mola syntax: # 01 Setup | Dimensions
  3. Add Mola Builder component to canvas
  4. Connect a Boolean Toggle to the Update input
  5. Toggle Update to True to generate files
  6. Click Launch Main button to open the interface

2 Structural Logic (The Foundation)

Before diving into individual components, understanding the container system is essential. This is how Mola organizes your interface.

The Core Syntax

To organize elements into the interface, simply Group them in Grasshopper and apply the Mola naming syntax.

Naming Syntax # [Order] [Tab Name] | [Section Name]
  • # - Defines the group as a Main Panel container
  • Order - (e.g., 01, 02, 10, 20) Determines the sequence. Use increments of 10 to allow future insertions
  • Tab Name - Grouping category (creates tabs in the interface)
  • Section Name - The collapsible accordion header

Example: # 01 Settings | Grid Parameters

This creates a tab called "Settings" with a collapsible section called "Grid Parameters".

Panel Types

Main Panel (#) The primary control interface. Contains all interactive inputs (sliders, toggles, buttons).

Syntax: # 01 Parameters | Geometry
Info Panel (#i) Secondary display panel for metrics, status, and read-only information.

Syntax: #i 01 Stats | Area Analysis

Section Modifiers

Symbol Effect Example
+ Section starts expanded # 01 +Important Settings
. Flat display (no collapsible frame) #i 01 .Live Status

Layout Sub-Groups

Within a main section, create sub-groups to modify layout flow:

Symbol Name Effect
= Row Places items side-by-side (Horizontal Flex)
- Divider Adds separator line. Name it - Label for text

3 Interface Elements

Now that you understand the container system, here are the individual components and how they render in the web interface.

Number Slider

Grasshopper Number Slider becomes a web slider with direct input capability.

  • Precision: Automatically detects decimal precision (integer, 0.1, 0.01) from GH settings
  • Input Box: Paired with numerical input for precise value entry
  • Range: Respects Min/Max limits defined in Grasshopper

Boolean Toggle

Grasshopper Boolean Toggle adapts based on its rendering context. The context is determined by the sub-group it belongs to.

Default Mode Renders as a checkbox styled with your accent color.

Context: No sub-group
Radio Logic (@) Mutually exclusive toggles - only one can be active at a time.

Context: Place toggles in sub-group named @
Grid System (*) Compact icon toolbar for efficient space usage.

Context: Place in sub-group named *
  • Add w for wrapping: * w
  • Add size: * w SMALL (TINY, SMALL, MEDIUM, LARGE)
Pill Switch (!) Modern iOS-style toggle switch.

Context: Place in sub-group named !

Button

Grasshopper Button Object for triggering actions. Rendering adapts to context.

Default Mode Full-width button.

Context: No sub-group
Grid Mode (*) Momentary icon button in compact toolbar.

Context: Place in sub-group named *

Value List

Grasshopper Value List adapts based on its List Mode setting (right-click component > List Mode):

Dropdown Menu GH Mode: Check List (default)
Standard HTML select dropdown for single selection.
Cycle Control GH Mode: Value Cycle
Left/Right arrows for sequential navigation.
Multi-Select Checklist GH Mode: Value Sequence
Checkbox list for selecting multiple items simultaneously.

Colour Swatch

Grasshopper Colour Swatch becomes a native OS color picker:

  • Full RGB/Hex selection support
  • Real-time updates
  • Visual color preview

Text Panel (Display)

Grasshopper Panel serves multiple display purposes:

Typography Modes

Rename the Panel object (right-click > Name) to change hierarchy:

  • Header 1: Name the panel #
  • Header 2: Name the panel ##
  • Header 3: Name the panel ###
  • Body Text: Leave name blank or standard

Rich Formatting

Panel content supports basic markdown:

  • **text** for Bold
  • __text__ for Italic
  • Line breaks preserved

Text Panel (Input)

Create editable text fields for user input:

  • Name the panel starting with quotation mark: " Project Name
  • Panel content becomes default value
  • User can overwrite in the interface

Icon Assignment

Mola automatically links components to image assets from the Mola_Assets folder:

  • Automatic: Component named "Calculate" loads Calculate.png
  • Manual Override: Component named [play] Calculate loads play.png but displays "Calculate"

4 Fixed Toolbars

Create persistent toolbars docked to screen edges for quick access to critical functions.

Syntax #t [POSITION] [SIZE] [LABELS]
Positions
  • TOP - Docked to top edge
  • LEFT - Docked to left edge
  • RIGHT - Docked to right edge
  • MENU - Context menu (right-click to open)
  • Multiple: Combine positions: #t TOP LEFT
Sizes & Labels Sizes:
  • TINY (24px), SMALL (28px), MEDIUM (32px)
  • LARGE (36px), XLARGE (40px)
  • Custom: Specify exact size: #t TOP 48
Labels: Add LABELS to show text under icons

Examples

  • #t TOP LARGE LABELS - Top toolbar, large icons, with text labels
  • #t LEFT MEDIUM - Left sidebar, medium icons, no labels
  • #ti MENU SMALL - Info panel context menu, small icons

5 Visual Styling (The Styler Component)

The Mola Styler component provides comprehensive theme control.

Built-in Presets

Number Name Description
0 Custom Use your own color/font settings
1 Analytics Blue, professional, clean
2 Coral Warm, soft, approachable
3 Mint Fresh, clean, minimal
4 Honey Warm, cozy, inviting
5 Soft Neumorphic, subtle depth
6 Brutalist Minimal, stark, monochrome
7 Bauhaus Bold, geometric, structured

Custom Styling

Set Preset to 0 (Custom) and adjust individual parameters:

  • Accent Color - Buttons, interactions, highlights
  • Page Color - Background
  • Card Color - Panel backgrounds
  • Text Color - Font color
  • Font Family - Typeface (e.g., "Inter", "Roboto")
  • Font Size - Base size (11-16px recommended)
  • Border Radius - Corner roundness (0-20px)
  • Spacing - 0=Compact, 1=Normal, 2=Relaxed
  • Shadow Level - Depth effect (0-5)

Dark Mode

Mola automatically detects system theme (light/dark). Optionally override:

  • Dark Mode: Enable/disable dark theme support
  • Dark Accent: Custom accent color for dark mode (optional)

Logo Integration

Add your company or project logo to the interface header for professional branding.

Setup Steps:

  1. Place your logo file in the Mola_Assets folder (next to your .gh file)
  2. In the Styler component, set Logo Name to your filename (e.g., "logo.png")
  3. Choose Logo Size: 0=Small, 1=Medium, 2=Large
  4. Connect Styler to Builder and update

Supported Formats: PNG, SVG, JPG (PNG recommended for transparency)

Best Practices: Use square images (1:1 ratio) for consistent appearance. Recommended resolution: 200x200px minimum.

6 The Builder Component

The Mola UI Builder is the core engine that scans your Grasshopper canvas and generates the web interface.

Inputs
  • Update - Toggle to rebuild the interface files
  • Design - Connect Styler output for custom theme
  • Layout - True for vertical tabs, False for horizontal tabs
  • Title - Application title (appears in header)
  • Logo - Logo filename from Mola_Assets folder
  • Width (Main/Info) - Panel widths in pixels
  • Height - Panel height in pixels
Outputs & Actions
  • Status - Connection status and active clients
  • Elements - List of all components: Type, Name, Value
Launch Buttons:
  • Launch Main - Opens Main panel
  • Launch Info - Opens Info panel

7 The Runner Component

The Mola Runner is an independent tool that launches and manages the web application without rebuilding files.

Use Cases

  • Running presentations or demos
  • Persistent server without canvas scanning
  • Cleaner workflow once interface is finalized
Inputs
  • Run App - Toggle to start/stop WebSocket server
  • Show Window - Button to force open/refresh windows
  • Always on Top - Keep windows above Rhino (Windows only)
  • Clear Positions - Reset saved window positions
  • Restart Server - Restart WebSocket connection
  • Port - Custom port number (default: 8181)
Outputs
  • Status - Server status and connected clients
  • Elements - Current component states: Type, Name, Value

Important: You must run the Builder's "Update" at least once to generate the app files before the Runner can launch them.

8 Cross-Platform Support

Mola UI provides full feature parity across Windows and Mac with platform-optimized implementations.

Windows
  • Single HTML file with hash routing (#M, #I)
  • Uses Fleck WebSocket library
  • Edge/Chrome app mode support (clean, toolbar-free windows)
  • Automatic window position saving
Mac
  • Separate HTML files (index_main.html, index_info.html)
  • Uses WebSocketSharp library
  • Chrome/Safari support with new window mode
  • Opens as browser tabs/windows (not app mode)

Universal Features

  • Real-time WebSocket updates
  • Dark mode auto-detection
  • Accent color theming
  • Identical UI rendering
  • Same syntax and features

9 Troubleshooting

No HTML file found Solution: Toggle Builder's "Update" input to generate files first.
Port 8181 busy Solution: Close Rhino completely and restart, or change port number in Runner component.
Components not appearing Check: Group name starts with # and components are inside the group.
Styling not applying Solution: Connect Styler to Builder BEFORE toggling Update.
Browser doesn't open Check: Firewall settings, or try a different browser (Chrome/Edge/Safari).
Changes not updating in browser Solution: Refresh browser (F5) or toggle Update again in Builder.

10 Technical Specifications

System Requirements

  • Grasshopper 1.0 or later
  • Windows 10/11 or macOS 10.14+
  • Modern web browser (Chrome, Edge, Safari, Firefox)

Network

  • WebSocket Server: Port 8181 (customizable)
  • Protocol: ws://localhost:8181
  • Real-time bidirectional communication

File Generation

  • Automatic HTML/CSS generation on Update
  • Theme CSS embedded in generated files
  • Window positions saved in window_positions.json (Windows)

Language Support

  • Right-to-Left (RTL) Languages: Automatic detection and support
  • Supports Hebrew, Arabic, Persian, and other RTL scripts
  • Interface layout automatically mirrors for RTL text
  • No configuration required - detects text direction from content
  • Mixed LTR/RTL content handled intelligently
